# 屏幕分类与前端 Canvas 开发注意点

在前端开发，尤其是 Canvas 绘图和图形渲染中，理解屏幕的差异至关重要。

## 1. 屏幕通常分类

### 标准分辨率屏幕 (Standard DPI / Standard Definition)
- **特征**: `window.devicePixelRatio` (DPR) 通常为 1。
- **代表**: 较旧的桌面显示器、部分入门级笔记本。
- **对开发影响**: 1个 CSS 像素对应 1个 物理像素，绘制最为简单，性能消耗最低。

### 高清屏 (High DPI / Retina 屏)
- **特征**: DPR > 1 (常见为 2, 3, 甚至 3.5)。
- **代表**: MacBook (Retina), iPhone, 高端 Android 手机, 4K/5K 显示器。
- **对开发影响**: 如果不进行适配，Canvas 绘制的内容会显得模糊，因为原本 1 个物理像素的位置被多个物理像素填充，导致了“插值模糊”。

### 超宽屏与超大屏 (Ultrawide / Large Screens)
- **特征**: 极大的分辨率 (如 3440x1440px 或更大) 或特殊纵横比 (21:9)。
- **对开发影响**: Canvas 画布面积巨大，导致渲染性能下降和内存占用激增。

### 移动端屏幕 (Mobile Screens)
- **特征**: DPR 变化范围广，且存在横竖屏切换 (Orientation Change)。
- **对开发影响**: 需要处理动态的视口变化。

## 2. 前端 Canvas 开发对不同屏幕的注意点

### 核心挑战：Retina 屏模糊问题
由于 Canvas 的 `width` 和 `height` 属性定义的是画布的**绘图表面大小**（像素数），而 CSS 样式定义的则是其在页面上的**显示大小**。在高清屏上，必须使绘图表面的像素数大于显示大小。

#### 解决方案：DPR 适配法
1. **获取当前 DPR**: `const dpr = window.devicePixelRatio || 1;`
2. **设置画布物理像素**: `canvas.width = displayWidth * dpr;`
3. **设置画布显示尺寸**: `canvas.style.width = displayWidth + 'px';`
4. **缩放上下文**: `ctx.scale(dpr, dpr);`

这样，后续所有的绘制指令（如 `ctx.fillRect(0, 0, 100, 100)`）都会自动乘以 DPR，确保在高清屏上依然清晰。

### 核心挑战：性能与内存管理
- **注意点**: 随着 DPR 的增加，Canvas 的总像素数是按平方增长的。
  - 一个 1000x1000 的画布在 DPR=2 时实际有 400 万像素。
  - 在 DPR=3 时则有 900 万像素。
- **建议**:
  - 对于不需要极端清晰的复杂动画，可以适当限制 DPR 的上限（例如：`Math.min(window.devicePixelRatio, 2)`）。
  - 在大屏幕上避免创建过大的 Canvas，考虑使用分层 Canvas (Layered Canvas) 或局部更新。

### 核心挑战：1px 线条清晰度
- **注意点**: 在 Canvas 中绘制 1px 线条时，如果坐标是整数（如 `10`），线条中心会对准像素边缘，导致向两侧各渲染 0.5px，产生模糊。
- **解决方案**: 将坐标偏移 0.5px（如 `10.5`），使其中心对准像素中心。

### 核心挑战：浏览器缩放 (Browser Zoom)
- **注意点**: 用户手动缩放网页时，`devicePixelRatio` 会发生变化。
- **建议**: 监听 `window.onresize` 事件或使用 `matchMedia` 监听 DPI 变化，实时重新调整 Canvas 的物理像素大小。
